Branches

Branches are used to isolate development work without affecting other branches in the repository.

Merge

The git merge command lets you take independent lines of development on separate branches and integrate them into a single branch.

Commit

The git commit command is similar to saving a file. A commit is a change to one or more files to your branch.

Tags

Tags are specific reference points in the git history. They are used to capture an individual point in your repository's history.

Repositories

A git repository is the .git/ folder inside a project. It tracks all changes made to files in the project, building a history over time.
